Creating a command

To create a command you will have to create a new .js file inside the commands/ folder.
This is what a default command looks like. 


exports.name = "name-cmd"
exports.level = 0;
exports.aliases = ["alias-1", "alias-2"]

exports.command = function(client, message) {

  // Execute code here.

}

Possible exports

  • .name
    TypeString
    Required

    Describes the name of the message function. Must be unique

  • .level
    TypeNumber / null
    Default0

    Level determines who can use this command, see the permission instructions for more information.

  • .ignoredChannels
    TypeArray (String) / null
    Default["dm", "group"]

    Code that will be executed before a command runs.

    Possible channel types:
    • "dm"
    • "group"
    • "text"

  • .komada
    TypeBoolean / null
    Defaultfalse

    Use this export if you're using a komada command file.

  • .command
    Typefunction
    Required

    • The code that will be executed on command execution.

    Arguments
    • client
      TypeClient Object
    • message
      TypeMessage Object

    Komada support Deprecated

    This framework does support Komada's command system, but it does not use any of their other systems. This feature will likely be removed at a near point in the future, when more features get added that Komada doesn't have / handles differently.

    Editor's note (31th of march 2018) | Komada support has been dropped in all versions following 0.0.6
