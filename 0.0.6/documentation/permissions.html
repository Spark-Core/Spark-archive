What are Permission files?

Permission files check a certain condition before running a command, to check if the user of the command has the permission to execute it. You specify this by adding the level export on the command corresponding to the level on the permission file. If you want to do something before a command executes but after the permission has been cleared, use a message function.

Default permissions:

Spark has default permission files added for easy startup, ofcourse you can overwrite them if you don't want to use them.

  • admin.js | Level 4

    Checks if the user has a role matching the admin role specified in your initial config adminRole: "role-name" If no role is specified or the user doesn't have a role like this the bot will not execute the command.

  • moderator.js | Level 3

    Checks if the user has a role matching the mod role specified in your initial config modRole: "role-name" If no role is specified or the user doesn't have a role like this the bot will not execute the command.

  • member.js | Level 0

    Always allows execution of the command.

  • owner.js | Level 10

    Allows execution of command if the command is run by the creator of the bot.

Adding permission files

exports.name = "name-here"
exports.level = 0
exports.filter = (client, message) => {


return false; // Allows execution of the commad.
return true;  // Stops execution of the command.

}

Possible exports

  • .name
    TypeString
    Required

    Describes the name of the Permission. Must be unique

  • .level
    TypeNumber
    Required

    This is the level which will get checked by the command file.

  • .filter
    Typefunction
    Required

    Code that will be executed to see if the user has permission. Must return a boolean (true for stopping executing, false for letting it go through)

    • Stopping commands from executing

    To stop a command from executing return true to make it stop executing, or false to let it continue.
